1Prepare the inputs

Atlas-Builder reads one folder. Inside it go your photos and a plain-text plan file. The folder can be named anything.

Name each photo by where it belongs

A photo's name tells the builder its species (a column) and its view (a row, like bark or leaves). Use either layout:

# one folder, dots between the parts
Ceiba.bark.jpg
Ceiba.leaves.jpg
Jacaranda.flowers.jpg

# or a subfolder per species
Ceiba/bark.jpg
Ceiba/leaves.jpg
Jacaranda/flowers.jpg
Spaces, underscores, and dashes all count as the same. Coast Redwood, Coast_Redwood, and Coast-Redwood land in the same column, so you don't have to be exact.

Write the plan file

A plain-text file — name it plan.txt — names the atlas and sets the order of columns and rows. The caption lines add the words shown beneath each species.

Atlas Name: Ajijic Tree Atlas
subtitle: Trees of the lakeshore

Column: Ceiba
Column: Jacaranda
Column: Laurel

Row: bark
Row: leaves
Row: flowers

Caption:
Ceiba: Ceiba pentandra — the kapok.
Jacaranda: Violet bloom in spring.
Laurel: Ficus, shade of the plaza.
Missing a photo for some cell? Leave it out, or write n/a — the builder shows a quiet placeholder instead of a gap.

Run Atlas-Builder

Open Atlas-Builder.html in your browser. Choose your folder, choose plan.txt, then click Build.

It resizes every photo and saves one file, Atlas.html, to your computer. That file carries all the photos inside it, so it opens even with no internet.

2Publish to GitHub

Your Atlas.html already works on your own machine. To give it a web address anyone can visit, host it free on GitHub Pages.

Make an account and a repository

Sign up at github.com. Click New to create a repository named AjijicTreeAtlas, set it Public, and check Add a README so it isn't empty.

Add your atlas file

Open the repo and choose Add file → Upload files, then drag in Atlas.html. If you have Git installed, you can instead push from the folder:

git add .
git commit -m "Add the atlas"
git push

Turn on GitHub Pages

In the repo, open Settings → Pages. Under Source, pick the main branch and the root folder, then Save.

Open your live atlas

A minute later it's online:

https://USERNAME.github.io/AjijicTreeAtlas/Atlas.html
Shorter address: rename Atlas.html to index.html and the link becomes just …github.io/AjijicTreeAtlas/.
Share that link with anyone, on any device. To update later, rebuild in Part 1 and re-upload the new file in Part 2.
